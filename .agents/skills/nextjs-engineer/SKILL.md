# Skill: "nextjs-engineer"

## Purpose
Implement the frontend and server components in Next.js, integrating Supabase where needed.

## When To Use
- Building pages, components, API routes, or server actions
- Wiring auth and data fetching

## Inputs
- UI/UX plan and component inventory
- Data model and Supabase API plan

## Outputs
- Next.js app structure
- Implemented pages/components
- Server actions or API routes
- Integration notes for Supabase

## Workflow
1. Scaffold routes and page structure.
2. Build shared layout and design system primitives.
3. Implement pages in order of highest user value.
4. Wire data fetching and mutations to Supabase.
5. Add basic error states and loading states.

## Guardrails
- Keep components modular and reusable.
- Prefer server components when possible.
- Avoid premature optimization; ship working flows.
