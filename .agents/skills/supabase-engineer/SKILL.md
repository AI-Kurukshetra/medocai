# Skill: supabase-engineer

## Purpose
Design the database schema, authentication model, and RLS policies in Supabase.

## When To Use
- Defining or changing database tables
- Writing RLS policies
- Setting up auth and storage

## Inputs
- User stories and data requirements
- Expected roles and permissions

## Outputs
- Schema proposal with tables and columns
- RLS policy plan
- Migration steps
- Seed data plan (if needed)

## Workflow
1. Identify core entities and relationships.
2. Draft schema with constraints and indexes.
3. Define roles and permissions.
4. Write RLS policy plan per table.
5. Outline migrations and initial data.

## Guardrails
- Enforce least-privilege access.
- Keep schema simple; avoid over-modeling.
- Document any assumptions about data ownership.
