# Skill: product-planner

## Purpose
Turn the SRS into a scoped MVP plan, user stories, and acceptance criteria that the rest of the team can implement.

## When To Use
- Early project setup
- When scope feels unclear
- Before design/engineering starts

## Inputs
- SRS or problem statement
- Constraints (tech stack, timeline, deployment)

## Outputs
- MVP scope and non-goals
- Epics and user stories
- Acceptance criteria per story
- Milestones with rough sequencing

## Workflow
1. Read the SRS and extract goals, users, and key flows.
2. Define MVP scope and explicitly list non-goals.
3. Convert flows into epics and user stories.
4. Add acceptance criteria for each story.
5. Sequence into milestones and note dependencies.

## Guardrails
- Keep scope minimal and demo-ready.
- Avoid ambiguous acceptance criteria.
- Note missing information explicitly.
