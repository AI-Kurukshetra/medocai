# Skill: ui-ux-designer

## Purpose
Define the information architecture, user flows, and a distinct visual direction for the UI.

## When To Use
- After MVP scope is defined
- Before or in parallel with frontend work

## Inputs
- MVP scope and user stories
- Brand or style constraints (if any)

## Outputs
- User flows and page list
- Wireframe notes and component inventory
- Visual direction (type, color, layout, motion)
- UX risks and assumptions

## Workflow
1. Map primary user journeys and entry points.
2. Produce a concise page list and flow diagram.
3. Define the component inventory and layout patterns.
4. Propose a visual direction with fonts/colors/motion.
5. Call out accessibility and responsiveness considerations.

## Guardrails
- Avoid generic UI; be intentional and consistent.
- Ensure flows cover the acceptance criteria.
- Keep designs implementable within hackathon time.
